MockPilot

A lightweight local mock service for frontend-backend collaboration · VS Code extension

Install / Start Features

Rapidly write, manage & run local Mock APIs inside VS Code

Based on project root .mock/ YAML configs: automatically registers routes, supports multiple response rules, hot reload & visual editing.

YAML Example
name: Get User Info
description: Get basic user info
method: GET
endpoint: /api/user
rules:
  - name: Normal Response
    status: 200
    headers:
      Content-Type: application/json
    body:
      id: 1
      name: Leo
    delay: 0

Core Features

Automatic Route Registration

Scans YAML files under .mock/, auto-generating local routes ready to use.

Multiple Response Rules

Define multiple rules (normal, error, delay, etc.) for the same route; switch with one click in the UI.

WebView Visual Editing

Open an editor panel in the VS Code sidebar to maintain YAML configs WYSIWYG.

Hot Reload & File Watching

Saving a YAML file triggers Mock Server reload for fast iteration.

Installation & Usage

  1. Open the project root folder in VS Code.
  2. Press Cmd+P, type Extensions: Install Extension, then search and install this extension (or load from source).
  3. Open the MockPilot panel in the sidebar and click Start (default port 9527).
  4. Place your YAML files into the project root .mock/ directory and make requests via http://localhost:9527.

Tip: You can modify port, auto-start and other settings in the extension preferences.
